model account {
  account_id        Int                      @id @default(autoincrement())

## 3. Міграція 1 — Додавання таблиці `profile` 

### Опис змін

- Додано нову модель `profile` для зберігання персональних даних гравця (`first_name`, `last_name`), пов'язану з `account` відношенням один-до-одного.

### Зміни у `schema.prisma`

**Після (`account` + нова модель `profile`):**
```prisma
model account {
  account_id        Int                      @id @default(autoincrement())
  email             String                   @unique @db.VarChar(255)
  account_password  String                   @db.VarChar(24)
  username          String                   @unique @db.VarChar(24)
  registration_date DateTime?                @db.Timestamp(6)
  online            online_status
  status            account_status
  in_game_time      Unsupported("interval")?
  is_premium        Boolean?                 @default(false)   // ← нове поле
  profile           profile?                                   // ← зворотній зв'язок
  game_character    game_character[]
}

// Нова таблиця
model profile {
  profile_id Int     @id @default(autoincrement())
  account_id Int     @unique
  first_name String
  last_name  String
  account    account @relation(fields: [account_id], references: [account_id])
}
```

## 6. Верифікація через Prisma Client

### Генерація клієнта

```bash
npx prisma generate
```

### Тестовий скрипт `index.js`

```javascript
const { PrismaClient } = require('@prisma/client');
const { PrismaPg } = require('@prisma/adapter-pg');
const { Pool } = require('pg');

require('dotenv').config();

const pool = new Pool({ connectionString: process.env.DATABASE_URL });
const adapter = new PrismaPg(pool);
const prisma = new PrismaClient({ adapter });

async function main() {
    const account = await prisma.account.create({
        data: {
            email:            'profile_test@kpi.ua',
            account_password: 'SecurePass123',
            username:         'ProfileUser',
            online:           'Online',
            status:           'OK',
            is_premium:       false,
            profile: {
                create: {
                    first_name: 'Ivan',
                    last_name:  'Student',
                },
            },
        },
    });

    const profiles = await prisma.profile.findMany({
        include: {
            account: true,
        },
    });

    console.log(profiles);
}

main()
    .catch(console.error)
    .finally(() => prisma.$disconnect());
```

**Запуск:**

```bash
node index.js
```

**Підсумок:**

У ході лабораторної роботи:

1. Встановлено та налаштовано Prisma ORM з адаптером `@prisma/adapter-pg` для підключення до існуючої бази даних PostgreSQL зі схемою з ЛР5.
2. Виконано інтроспекцію схеми (`db pull`), яка автоматично згенерувала моделі Prisma для всіх таблиць.
3. Застосовано три послідовні міграції — кожна зафіксована в окремій підпапці `prisma/migrations/` із відповідним SQL-файлом, що забезпечує повне версіонування схеми.
4. Верифіковано коректність змін через тестовий скрипт `index.js`: успішно створено акаунт із вкладеним профілем і отримано дані через `findMany` з `include`.

Prisma ORM спрощує управління схемою порівняно з ручним написанням DDL: зміни оголошуються декларативно у `schema.prisma`, SQL генерується автоматично, а вся історія міграцій зберігається у Git.
