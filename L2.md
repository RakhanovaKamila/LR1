## 1. Опис схеми

Схема описує багатокористувацьку рольову гру (MMORPG). Вона охоплює акаунти гравців, ігрові сервери, персонажів, інвентар, предмети, зброю, регіони, будівлі та квестову систему.

---

## 2. Перелік таблиць та їх структура

### Таблиця 1 — Account

Зберігає інформацію про облікові записи гравців.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| account_id | SERIAL | PK | Унікальний ідентифікатор |
| email | VARCHAR(255) | NOT NULL, UNIQUE | Електронна пошта |
| account_password | VARCHAR(24) | NOT NULL, CHECK (LENGTH >= 8) | Пароль (мін. 8 символів) |
| username | VARCHAR(24) | NOT NULL, UNIQUE | Ігровий псевдонім |
| registration_date | TIMESTAMP | — | Дата реєстрації |
| online | online_status | NOT NULL | Статус присутності |
| status | account_status | NOT NULL | Статус акаунту |
| in_game_time | INTERVAL | — | Загальний час у грі |

**Типи ENUM:** `online_status` ('Online', 'Offline', 'Sleeping'), `account_status` ('Banned', 'OK', 'Muted', 'Penalized')

---

### Таблиця 2 — game_server

Ігрові сервери, до яких підключаються гравці.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| server_id | SERIAL | PK | Унікальний ідентифікатор |
| server_name1 | VARCHAR(64) | NOT NULL | Назва сервера |
| ip_address | VARCHAR(45) | NOT NULL | IP-адреса |
| status | server_status | NOT NULL | Стан сервера |
| current_characters | INT | DEFAULT 0, CHECK >= 0 | Поточна кількість гравців |

**Тип ENUM:** `server_status` ('On', 'Off')

---

### Таблиця 3 — Region

Регіони, розміщені на серверах.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| region_id | SERIAL | PK | Унікальний ідентифікатор |
| server_id | INT | NOT NULL, FK → game_server | Сервер регіону |
| region_name | VARCHAR(100) | — | Назва регіону |
| region_biome | VARCHAR(100) | — | Тип біому |
| region_min_level | INT | NOT NULL | Мінімальний рівень для входу |

---

### Таблиця 4 — Game_Structure

Будівлі та споруди у регіонах.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| structure_id | SERIAL | PK | Унікальний ідентифікатор |
| region_id | INT | FK → Region | Регіон будівлі |
| structure_name | VARCHAR(50) | — | Назва |
| structure_position | DECIMAL(12,4) | NOT NULL, DEFAULT 0 | Позиція на карті |
| structure_type | structure_type1 | NOT NULL | Тип будівлі |
| is_buyable | BOOLEAN | — | Чи можна купити |
| structure_cost | DECIMAL(10,2) | DEFAULT 0 | Вартість |

**Тип ENUM:** `structure_type1` ('ruin', 'circus', 'church', 'mine')

---

### Таблиця 5 — Game_Character

Ігрові персонажі, прив'язані до акаунтів.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| character_id | SERIAL | PK | Унікальний ідентифікатор |
| account_id | INT | FK → Account | Власник персонажа |
| server_id | INT | FK → game_server | Сервер персонажа |
| region_id | INT | FK → Region | Поточний регіон |
| respawn_structure_id | INT | FK → Game_Structure | Точка відродження |
| nickname | VARCHAR(50) | — | Ім'я персонажа |
| character_level | INT | DEFAULT 0, CHECK (0..100) | Рівень |
| character_health | INT | DEFAULT 10 | Здоров'я |
| character_stamina | INT | DEFAULT 10 | Витривалість |
| character_class | class_type | — | Клас персонажа |
| character_position | DECIMAL(12,4) | — | Позиція на карті |

**Тип ENUM:** `class_type` ('Mage', 'Warrior', 'Archer', 'Berserker')

---

### Таблиця 6 — ItemTemplate

Шаблони предметів (статичні дані).

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| item_id | SERIAL | PK | Унікальний ідентифікатор |
| item_name | VARCHAR(50) | NOT NULL | Назва предмета |
| item_type | item_category | NOT NULL | Категорія |
| item_description | TEXT | — | Опис |
| item_weight | DECIMAL(5,2) | DEFAULT 0.0 | Вага |
| item_max_stack | INT | CHECK (1..64) | Максимальний стек |
| rarity | rarity_type | NOT NULL | Рідкість |
| item_cost | DECIMAL(10,2) | — | Базова вартість |

**Типи ENUM:** `item_category` ('weapon', 'resource', 'trash'), `rarity_type` ('Common', 'Uncommon', 'Rare', 'Legendary')

---

### Таблиця 7 — Inventory_Slot

Інвентар персонажа — слоти з предметами.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| slot_id | SERIAL | PK | Унікальний ідентифікатор слоту |
| character_id | INT | FK → Game_Character | Власник слоту |
| item_id | INT | FK → ItemTemplate | Предмет у слоті |
| item_amount | INT | DEFAULT 1, CHECK > 0 | Кількість |
| is_equipped | BOOLEAN | DEFAULT FALSE | Чи екіпіровано |

---

### Таблиця 8 — WeaponStas

Бойові характеристики зброї.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| item_id | INT | PK, FK → ItemTemplate | Посилання на предмет |
| damage | DECIMAL(10,1) | — | Шкода |
| weapon_bonus | bonus_type | — | Бонусний ефект |

**Тип ENUM:** `bonus_type` ('Strength', 'Accuracy')

---

### Таблиця 9 — Quest_System

Квести та їх нагороди.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| quest_id | SERIAL | PK | Унікальний ідентифікатор |
| item_id | INT | NOT NULL, FK → ItemTemplate | Предмет-нагорода |
| quest_name | VARCHAR(50) | — | Назва квесту |
| quest_desciription | TEXT | — | Опис |
| quest_xp | DECIMAL(10,2) | — | Нагорода XP |

---

### Таблиця 10 — Character_Quests

Зв'язкова таблиця M:N між персонажами та квестами.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| character_id | INT | PK (частина), FK → Game_Character | Персонаж |
| quest_id | INT | PK (частина), FK → Quest_System | Квест |
| quest_status | quest_status_type | NOT NULL | Стан квесту |

**Тип ENUM:** `quest_status_type` ('Active', 'Completed', 'Failed')

Складений первинний ключ `(character_id, quest_id)` гарантує, що один персонаж не може мати один квест двічі.

---

### Таблиця 11 — Structure_ownership

Зв'язкова таблиця M:N між персонажами та будівлями.

| Стовпець | Тип | Обмеження | Опис |
|----------|-----|-----------|------|
| character_id | INT | FK → Game_Character | Персонаж |
| structure_id | INT | FK → Game_Structure | Будівля |
| structure_role | structure_role_type | — | Роль у будівлі |

**Тип ENUM:** `structure_role_type` ('Owner', 'Co_owner', 'Guest')

## 3. DDL-скрипт (CREATE TABLE)

```sql
-- DROP ALL
DROP TABLE IF EXISTS Character_Quests CASCADE;
DROP TABLE IF EXISTS Inventory_Slot CASCADE;
DROP TABLE IF EXISTS WeaponStas CASCADE;
DROP TABLE IF EXISTS Quest_System CASCADE;
DROP TABLE IF EXISTS Game_Structure CASCADE;
DROP TABLE IF EXISTS Game_Character CASCADE;
DROP TABLE IF EXISTS Region CASCADE;
DROP TABLE IF EXISTS ItemTemplate CASCADE;
DROP TABLE IF EXISTS game_server CASCADE;
DROP TABLE IF EXISTS Account CASCADE;
DROP TABLE IF EXISTS Structure_ownership CASCADE;

DROP TYPE IF EXISTS quest_status_type CASCADE;
DROP TYPE IF EXISTS class_type CASCADE;
DROP TYPE IF EXISTS structure_type1 CASCADE;
DROP TYPE IF EXISTS bonus_type CASCADE;
DROP TYPE IF EXISTS rarity_type CASCADE;
DROP TYPE IF EXISTS item_category CASCADE;
DROP TYPE IF EXISTS server_status CASCADE;
DROP TYPE IF EXISTS account_status CASCADE;
DROP TYPE IF EXISTS online_status CASCADE;
DROP TYPE IF EXISTS structure_role_type CASCADE;

-- TYPES
CREATE TYPE online_status       AS ENUM ('Online', 'Offline', 'Sleeping');
CREATE TYPE account_status      AS ENUM ('Banned', 'OK', 'Muted', 'Penalized');
CREATE TYPE server_status       AS ENUM ('On', 'Off');
CREATE TYPE item_category       AS ENUM ('weapon', 'resource', 'trash');
CREATE TYPE rarity_type         AS ENUM ('Common', 'Uncommon', 'Rare', 'Legendary');
CREATE TYPE bonus_type          AS ENUM ('Strength', 'Accuracy');
CREATE TYPE structure_type1     AS ENUM ('ruin', 'circus', 'church', 'mine');
CREATE TYPE class_type          AS ENUM ('Mage', 'Warrior', 'Archer', 'Berserker');
CREATE TYPE quest_status_type   AS ENUM ('Active', 'Completed', 'Failed');
CREATE TYPE structure_role_type AS ENUM ('Owner', 'Co_owner', 'Guest');

-- TABLES
CREATE TABLE Account (
    account_id        SERIAL PRIMARY KEY,
    email             VARCHAR(255) NOT NULL UNIQUE,
    account_password  VARCHAR(24) NOT NULL CHECK (LENGTH(account_password) >= 8),
    username          VARCHAR(24) NOT NULL UNIQUE,
    registration_date TIMESTAMP,
    online            online_status NOT NULL,
    status            account_status NOT NULL,
    in_game_time      INTERVAL
);

CREATE TABLE game_server (
    server_id          SERIAL PRIMARY KEY,
    server_name1       VARCHAR(64) NOT NULL,
    ip_address         VARCHAR(45) NOT NULL,
    status             server_status NOT NULL,
    current_characters INT DEFAULT 0 CHECK (current_characters >= 0)
);

CREATE TABLE Region (
    region_id        SERIAL PRIMARY KEY,
    server_id        INT NOT NULL REFERENCES game_server(server_id),
    region_name      VARCHAR(100),
    region_biome     VARCHAR(100),
    region_min_level INT NOT NULL
);

CREATE TABLE Game_Structure (
    structure_id       SERIAL PRIMARY KEY,
    region_id          INT REFERENCES Region(region_id),
    structure_name     VARCHAR(50),
    structure_position DECIMAL(12, 4) NOT NULL DEFAULT 0.0000,
    structure_type     structure_type1 NOT NULL,
    is_buyable         BOOLEAN,
    structure_cost     DECIMAL(10, 2) DEFAULT 0
);

CREATE TABLE Game_Character (
    character_id         SERIAL PRIMARY KEY,
    account_id           INT REFERENCES Account(account_id),
    server_id            INT REFERENCES game_server(server_id),
    region_id            INT REFERENCES Region(region_id),
    respawn_structure_id INT REFERENCES Game_Structure(structure_id),
    nickname             VARCHAR(50),
    character_level      INT DEFAULT 0 NOT NULL CHECK (character_level >= 0 AND character_level <= 100),
    character_health     INT DEFAULT 10,
    character_stamina    INT DEFAULT 10,
    character_class      class_type,
    character_position   DECIMAL(12, 4)
);

CREATE TABLE ItemTemplate (
    item_id          SERIAL PRIMARY KEY,
    item_name        VARCHAR(50) NOT NULL,
    item_type        item_category NOT NULL,
    item_description TEXT,
    item_weight      DECIMAL(5, 2) DEFAULT 0.0,
    item_max_stack   INT CHECK (item_max_stack > 0 AND item_max_stack <= 64),
    rarity           rarity_type NOT NULL,
    item_cost        DECIMAL(10, 2)
);

CREATE TABLE Inventory_Slot (
    slot_id      SERIAL PRIMARY KEY,
    character_id INT REFERENCES Game_Character(character_id),
    item_id      INT REFERENCES ItemTemplate(item_id),
    item_amount  INT DEFAULT 1 CHECK (item_amount > 0),
    is_equipped  BOOLEAN DEFAULT FALSE
);

CREATE TABLE WeaponStas (
    item_id      INT PRIMARY KEY REFERENCES ItemTemplate(item_id),
    damage       DECIMAL(10, 1),
    weapon_bonus bonus_type
);

CREATE TABLE Quest_System (
    quest_id           SERIAL PRIMARY KEY,
    item_id            INT NOT NULL REFERENCES ItemTemplate(item_id),
    quest_name         VARCHAR(50),
    quest_desciription TEXT,
    quest_xp           DECIMAL(10, 2)
);

CREATE TABLE Character_Quests (
    character_id INT REFERENCES Game_Character(character_id),
    quest_id     INT REFERENCES Quest_System(quest_id),
    quest_status quest_status_type NOT NULL,
    PRIMARY KEY (character_id, quest_id)
);

CREATE TABLE Structure_ownership (
    character_id   INT REFERENCES Game_Character(character_id),
    structure_id   INT REFERENCES Game_Structure(structure_id),
    structure_role structure_role_type
);
```

## 6. Висновки

У ході лабораторної роботи була розроблена реляційна схема для бази даних багатокористувацької гри. Схема містить 11 таблиць, які відображають ER-модель проекту.

Для кожної таблиці визначені первинні ключі, зовнішні ключі для забезпечення цілісності посилань, обмеження `NOT NULL` на обов'язкових полях, `CHECK` для валідації числових діапазонів (рівень персонажа 0–100, розмір стека 1–64, мінімальна довжина пароля 8 символів) та `DEFAULT` для полів зі стандартними значеннями. Зв'язки M:N між персонажами і квестами та між персонажами і будівлями реалізовані через окремі зв'язкові таблиці зі складеним первинним ключем. Усі тестові дані успішно вставлені без порушення жодного обмеження.
