## 2. Блок 1: Отримання даних (SELECT)

### Запит 1.1 — Список акаунтів зі статусом `'OK'`

```sql
SELECT account_id, username, email, online
FROM Account
WHERE status = 'OK';
```

**Результат:** Вивід `account_id`, `username`, `email` та `online` для активних гравців. 

---

### Запит 1.2 — Персонажі рівня 10+, відсортовані за спаданням

```sql
SELECT character_id, nickname, character_level, character_class
FROM Game_Character
WHERE character_level >= 10
ORDER BY character_level DESC;
```

**Результат:** Відфільтрований список топ-персонажів. 

---

### Запит 1.3 — Екіпіровані предмети конкретного персонажа (`id = 1`)

```sql
SELECT inv.slot_id, it.item_name, it.rarity, it.item_type
FROM Inventory_Slot inv
JOIN ItemTemplate it USING (item_id)
WHERE inv.character_id = 1
  AND inv.is_equipped = TRUE;
```

**Мета:** Практика використання `JOIN` між `Inventory_Slot` та `ItemTemplate`. 

---

### Запит 1.4 — Зброя рідкості `'Rare'` або `'Legendary'`

```sql
SELECT item_id, item_name, rarity, item_cost
FROM ItemTemplate
WHERE item_type = 'weapon'
  AND rarity IN ('Rare', 'Legendary')
ORDER BY item_cost DESC;
```

**Результат:** Список найціннішої зброї, відсортований за ціною. 

---

### Запит 1.5 — JOIN трьох таблиць: персонажі, акаунти, регіони

```sql
SELECT
    gc.nickname,
    gc.character_level,
    gc.character_class,
    a.username,
    r.region_name,
    r.region_biome
FROM Game_Character gc
JOIN Account a USING (account_id)
JOIN Region r USING (region_id);
```


---

### Запит 1.6 — Активні квести персонажів

```sql
SELECT
    gc.nickname,
    qs.quest_name,
    qs.quest_xp,
    cq.quest_status
FROM Character_Quests cq
JOIN Game_Character gc USING (character_id)
JOIN Quest_System qs USING (quest_id)
WHERE cq.quest_status = 'Active';
```


---

### Запит 1.7 — Будівлі, доступні для купівлі, вартістю < 2000

```sql
SELECT structure_id, structure_name, structure_type, structure_cost
FROM Game_Structure
WHERE is_buyable = TRUE
  AND structure_cost < 2000;
```


---

### Запит 1.8 — Головні власники будівель (роль `'Owner'`)

```sql
SELECT
    gc.nickname,
    gs.structure_name,
    gs.structure_type,
    so.structure_role
FROM Structure_ownership so
JOIN Game_Character gc USING (character_id)
JOIN Game_Structure gs USING (structure_id)
WHERE so.structure_role = 'Owner';
```


---

### Запит 1.9 — Зброя з бонусом до `'Strength'`

```sql
SELECT
    it.item_name,
    it.rarity,
    ws.damage,
    ws.weapon_bonus
FROM WeaponStas ws
JOIN ItemTemplate it USING (item_id)
WHERE ws.weapon_bonus = 'Strength'
ORDER BY ws.damage DESC;
```


---

### Запит 1.10 — Акаунти `'Online'` зі статусом `'OK'`

```sql
SELECT username, email, online, in_game_time
FROM Account
WHERE online = 'Online'
  AND status = 'OK';
```

## 4. Блок 3: Зміна існуючих даних (UPDATE)

> **Правило безпеки:** Кожен `UPDATE` містить обов'язковий `WHERE`, щоб уникнути зміни всіх рядків таблиці. Перед та після оновлення виконувався контрольний `SELECT`.

### 3.1 — Підвищення рівня персонажа (`id = 1`)

```sql
UPDATE Game_Character
SET character_level = character_level + 5
WHERE character_id = 1;
```

**Результат:** Змінився лише один конкретний рядок. 

---

### 3.2 — Бан гравця (`account_id = 3`)

```sql
UPDATE Account
SET status = 'Banned', online = 'Offline'
WHERE account_id = 3;
```


---

### 3.3 — Виконання квесту (`character_id = 2`, `quest_id = 3`)

```sql
UPDATE Character_Quests
SET quest_status = 'Completed'
WHERE character_id = 2 AND quest_id = 3;
```


---

### 3.4 — Зміна ціни предмета `'Steel Shield'`

```sql
UPDATE ItemTemplate
SET item_cost = 200.00
WHERE item_name = 'Steel Shield';
```


---

### 3.5 — Переміщення персонажа в новий регіон (`character_id = 3`)

```sql
UPDATE Game_Character
SET region_id = 1
WHERE character_id = 3;
```

## 5. Блок 4: Видалення даних (DELETE)

>  **Правило безпеки:** Усі `DELETE` використовують `WHERE`. Перед видаленням виконувався перевірочний `SELECT`.

### 4.1 — Видалення провального квесту (`character_id = 3`, `quest_id = 4`)

```sql
DELETE FROM Character_Quests
WHERE character_id = 3 AND quest_id = 4;
```


---

### 4.2 — Видалення предмета з інвентарю (`character_id = 5`, `item_id = 7`)

```sql
DELETE FROM Inventory_Slot
WHERE character_id = 5 AND item_id = 7;
```


---

### 4.3 — Скасування гостьового доступу до будівлі

```sql
DELETE FROM Structure_ownership
WHERE character_id = 4 AND structure_id = 4;
```


---

### 4.4 — Каскадне видалення акаунта `'DragonBorn'`

Послідовно видаляються дочірні записи, а наприкінці — сам акаунт:

```sql
DELETE FROM Character_Quests WHERE character_id = 6;
DELETE FROM Inventory_Slot   WHERE character_id = 6;
DELETE FROM Game_Character   WHERE account_id = 6;
DELETE FROM Account          WHERE username = 'DragonBorn';
```

**Результат:** Контрольний `SELECT` показав повну відсутність даних про користувача, що підтверджує коректну роботу реляційної моделі.

## 6. Висновки

| Критерій | Результат |
|---|---|
| **Правильність** | Синтаксис SQL коректний, усі DML-запити виконано в PostgreSQL без помилок |
| **Ефективність** | Використовуються `JOIN` для зв'язної інформації та точні критерії фільтрації |
| **Безпека** | Усі `UPDATE` та `DELETE` містять `WHERE`; застосовано попереднє тестування через `SELECT` |
| **Документація** | SQL-скрипти містять коментарі, результати організовані логічними блоками |
