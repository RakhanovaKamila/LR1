Мета роботиМета лабораторної роботи — набути практичних навичок виконання OLTP-операцій у PostgreSQL: написати SQL-запити типу SELECT для отримання даних, INSERT для додавання нових записів, UPDATE для зміни існуючих та DELETE для безпечного видалення рядків. Операції виконуються на базі даних MMORPG-системи, розробленої в лабораторних роботах 1–2.2. Опис бази данихБаза даних моделює MMORPG-систему та містить 11 таблиць. Таблиця Account зберігає облікові записи гравців (email, username, статус, час у грі). Таблиця game_server містить інформацію про ігрові сервери (назва, IP, статус, кількість персонажів), а таблиця Region — регіони на цих серверах (біом, мінімальний рівень входу). Таблиця Game_Structure описує ігрові будівлі (тип, позиція, ціна). Таблиця Game_Character містить дані про персонажів гравців (рівень, клас, HP, стамінa).Додатково, таблиця ItemTemplate зберігає шаблони предметів (тип, рідкість, вага, вартість), а таблиця Inventory_Slot — інвентар персонажів (предмет, кількість, екіпіровка). Таблиця WeaponStas визначає характеристики зброї (урон, бонусний атрибут). Таблиця Quest_System описує квести (нагорода у вигляді предмета, досвід), таблиця Character_Quests веде журнал квестів персонажів (статус виконання). Права власності на будівлі (Owner/Co_owner/Guest) фіксуються у таблиці Structure_ownership.Таблиці використовують ENUM-типи для обмеження значень у ключових полях (online_status, account_status, class_type, rarity_type тощо) та зовнішні ключі для забезпечення цілісності даних.3. SELECT — отримання данихУ цьому блоці виконано 10 запитів на читання даних. Запити охоплюють прості фільтри (WHERE), сортування (ORDER BY), вибірку з кількох таблиць через JOIN, а також використання операторів IN та порівняння булевих значень.1.1. Список акаунтів зі статусом OKМета: Отримати логін, email та статус онлайн для всіх акаунтів без обмежень.SELECT account_id, username, email, online
FROM Account
WHERE status = 'OK';
Результат: Повернуто 2 рядки: акаунти Stalker та Merlin.1.2. Персонажі рівня 10+, впорядковані за рівнемМета: Знайти всіх досвідчених персонажів, відсортованих від вищого рівня до нижчого.SELECT character_id, nickname, character_level, character_class
FROM Game_Character
WHERE character_level >= 10
ORDER BY character_level DESC;
Результат: Повернуто 4 рядки: Guts (80), Dumbledore (50), RobinHood (25), Geralt (15).1.3. Екіпіровані предмети персонажа (Геральт)Мета: Отримати список предметів з інвентарю персонажа з id=1, які наразі екіпіровані.SELECT inv.slot_id, it.item_name, it.rarity, it.item_type
FROM Inventory_Slot inv
JOIN ItemTemplate it USING (item_id)
WHERE inv.character_id = 1
  AND inv.is_equipped = TRUE;
Результат: Повернуто 1 рядок: Iron Sword (Common, weapon).1.4. Зброя рідкості Rare або LegendaryМета: Знайти дорогоцінні предмети типу weapon для потреб торгівлі або балансування гри.SELECT item_id, item_name, rarity, item_cost
FROM ItemTemplate
WHERE item_type = 'weapon'
  AND rarity IN ('Rare', 'Legendary')
ORDER BY item_cost DESC;
Результат: Повернуто 3 рядки: Legendary Blade (5000), Heavy Axe (300), Magic Staff (250).1.5. Персонажі з акаунтами та регіонами (JOIN)Мета: Отримати зведену таблицю: нікнейм, рівень, клас, логін власника та регіон перебування.SELECT gc.nickname, gc.character_level, gc.character_class,
       a.username, r.region_name, r.region_biome
FROM Game_Character gc
JOIN Account a USING (account_id)
JOIN Region r USING (region_id);
Результат: Повернуто 5 рядків зі зведеними даними всіх персонажів.1.6. Активні квести персонажівМета: Знайти всі квести зі статусом Active разом із назвою квесту та нікнеймом персонажа.SELECT gc.nickname, qs.quest_name, qs.quest_xp, cq.quest_status
FROM Character_Quests cq
JOIN Game_Character gc USING (character_id)
JOIN Quest_System qs USING (quest_id)
WHERE cq.quest_status = 'Active';
Результат: Повернуто 3 рядки: активні квести Geralтa, Dumbledore та Guts.1.7. Будівлі на продаж дешевше 2000Мета: Знайти доступні для придбання споруди в межах бюджету початківця.SELECT structure_id, structure_name, structure_type, structure_cost
FROM Game_Structure
WHERE is_buyable = TRUE
  AND structure_cost < 2000;
Результат: Повернуто 2 рядки: Old Ruin Base (1000), Starter Ruin (100).1.8. Власники будівель (роль Owner)Мета: Отримати список персонажів, які є повноправними власниками споруд.SELECT gc.nickname, gs.structure_name, gs.structure_type, so.structure_role
FROM Structure_ownership so
JOIN Game_Character gc USING (character_id)
JOIN Game_Structure gs USING (structure_id)
WHERE so.structure_role = 'Owner';
Результат: Повернуто 3 рядки: Geralt, RobinHood, LittleTimmy.1.9. Зброя з бонусом StrengthМета: Знайти всі предмети з бонусом до сили разом із характеристиками урону.SELECT it.item_name, it.rarity, ws.damage, ws.weapon_bonus
FROM WeaponStas ws
JOIN ItemTemplate it USING (item_id)
WHERE ws.weapon_bonus = 'Strength'
ORDER BY ws.damage DESC;
Результат: Повернуто 3 рядки: Legendary Blade (99.9), Heavy Axe (35.0), Iron Sword (15.0).1.10. Акаунти Online без обмеженьМета: Знайти гравців, що зараз у грі та не мають жодних покарань.SELECT username, email, online, in_game_time
FROM Account
WHERE online = 'Online'
  AND status = 'OK';
Результат: Повернуто 1 рядок: Stalker.4. INSERT — додавання нових данихУ цьому блоці додано нові записи до 5 таблиць. Після кожної операції виконувався SELECT для підтвердження коректності вставки. Результат: Запис з'явився в Inventory_Slot для character_id=6, is_equipped=TRUE.2.5. Квест до журналу DragonBornCharМета: Призначити стартовий квест "First Blood" новому персонажу.INSERT INTO Character_Quests (character_id, quest_id, quest_status)

Результат: Запис з'явився в Character_Quests зі статусом Active.5. UPDATE — оновлення данихУ цьому блоці змінено існуючі записи в 3 таблицях. Кожен оператор UPDATE містить умову WHERE для точного вибору рядків. Перед кожним UPDATE виконувався попередній SELECT для перевірки поточного стану — відповідно до рекомендацій безпечної роботи з БД.3.1. Підвищення рівня персонажа GeraltМета: Нарахувати +5 рівнів персонажу character_id=1 після завершення квесту.UPDATE Game_Character
SET character_level = character_level + 5
WHERE character_id = 1;
Результат: character_level змінився з 15 на 20. SELECT до і після підтвердив зміну.3.2. Бан токсичного акаунта (DarkLord)Мета: Встановити статус Banned та примусово перевести акаунт в Offline.UPDATE Account

ВисновкиУ ході лабораторної роботи було виконано 24 SQL-запити (10 SELECT, 5 INSERT, 5 UPDATE, 4 DELETE) на базі даних MMORPG-системи в PostgreSQL.В ході роботи підтверджено, що оператор SELECT дозволяє гнучко вибирати дані з однієї або кількох таблиць завдяки JOIN, фільтрувати їх через WHERE та впорядковувати через ORDER BY. Оператор INSERT дотримується обмежень таблиці (NOT NULL, CHECK, FOREIGN KEY) і вимагає правильного порядку вставки при наявності залежностей між таблицями.Також було з'ясовано, що UPDATE без умови WHERE змінив би всі рядки таблиці, тому перед кожним UPDATE доцільно виконувати попередній SELECT для перевірки. Аналогічно, DELETE без WHERE очистив би таблицю повністю, а при видаленні пов'язаних записів необхідно суворо дотримуватись порядку: спочатку дочірні таблиці, потім батьківські.Використання ENUM-типів у схемі забезпечує валідацію на рівні БД, тому спроба вставити некоректне значення одразу завершується помилкою PostgreSQL. Усі запити були виконані успішно без синтаксичних або логічних помилок, а результати кожної операції підтверджено перевірним SELECT.
